# Game Theory & Negotiation — 게임이론 협상 적용

## Nash Equilibrium — 내쉬 균형

### 정의

"모든 참가자가 자신의 현재 전략 하에서 다른 선택을 통해 더 나은 결과를 얻을 수 없는 상태."

### 협상에서의 의미

| 개념 | 협상 사례 | 문제 |
|------|---------|------|
| **Pure Nash** (명확한 균형) | 시장 가격 = $100만. 양측 모두 수용 | 효율적이지만, 분배는 일방적 (판매자 우위) |
| **Mixed Nash** (확률적 균형) | "50%는 이 조건, 50%는 저 조건" | 불안정. 실제 협상에서 거의 나타나지 않음 |
| **Mutually Suboptimal** (상호 열등) | 양측이 강하게 고착 → 합의 불가 → 모두 손실 (Prisoner's Dilemma) | **돌파 기법의 핵심**: Nash로 가두어진 나쁜 균형을 벗어나기 |

### Prisoner's Dilemma in Negotiation

```
        상대: 협력   |  상대: 배신
우리: 협력  | (좋음, 좋음) | (나쁨, 최고)
우리: 배신  | (최고, 나쁨) | (최악, 최악)
```

- **개별 합리성**: 상대가 뭘 하든 "배신"이 최적 (최고 또는 최악 회피)
- **집단 합리성**: 양측 협력이 "좋음"으로 모두에게 최고
- **결과**: 양측 배신 → 최악의 균형에 도달

**협상에서의 교착**: "상대가 먼저 양보하면 나도 양보하겠지만, 상대가 강하게 나오면 나도 강하게"라는 상호 불신 → Prisoner's Dilemma 같은 최악의 균형.

### 해결책

1. **양측의 미래 관계 명시**: "우리는 이번뿐만 아니라 다음에도 함께 합니다" → Folk Theorem 적용 (후술)
2. **제3자 감시**: 계약 위반 시 구제 메커니즘 (법적, 평판 손상)
3. **신뢰 구축 제스처**: 자신부터 작은 양보 → 상대의 협력 신호 유도

## Folk Theorem — 민속정리

### 핵심

**반복되는 게임(Infinitely Repeated Game)에서는, 충분한 미래 가치(Discount Factor δ > 일정 수준)가 있으면, 양측 협력이 내쉬 균형이 된다. 구속력 있는 계약 없이도.**

### 수학적 직관

```
현재 협력 → 미래 협력 (높은 가치)
현재 배신 → 미래 영구적 배신 (낮은 가치)

현재가치: 협력 보상 + δ × (미래 협력 가치) > 배신 보상 + δ × (미래 배신 가치)

δ가 충분하면, 협력이 합리적.
```

### 협상에서의 적용

| 상황 | 할인인수 δ | 협력 가능성 | 전술 |
|------|-----------|-----------|------|
| **일회성 거래** (M&A, 인수·합병) | δ ≈ 0 (미래 무관) | 낮음. Nash = 상호 의심 | 구속력 있는 계약 + 제3자 감시 필수 |
| **반복 거래** (공급업체, 장기 파트너십) | δ > 0.7 (미래 중요) | 높음. 협력이 균형 | 향후 거래 명시 + 신뢰 신호 강조 |
| **불명확한 미래** (불황, 산업 변화) | δ ≈ 0.3~0.5 (미래 불확실) | 중간. 부분 협력 | 단계적 commitment (contingent contracts) |

### 실전 체크

**"이번 협상이 끝나면 우리가 다시 만날 가능성이 얼마나 되는가?"**

- 가능성 높음 (미래 가치 높음) → Folk Theorem 작동 → "우리가 함께 성공하는 게 우리 둘 다에게 이득"이라는 메시지 강조
- 가능성 낮음 (일회성) → Nash로 인한 의심 고착 가능 → 구속력 있는 설계 필수

## Information Asymmetry — 정보 비대칭

### 정의

한쪽이 다른 쪽보다 더 많은 정보를 보유한 상태. 신뢰·만족도·성과에 악영향.

### 협상에서의 3대 비대칭

| 비대칭 | 숨기는 쪽 | 영향 | 전술 |
|-------|---------|------|------|
| **비용 정보** | 판매자가 진짜 비용을 숨김 | 수익률 조작. 구매자 overpay | "원가를 공개해야 신뢰를 회복할 수 있다" |
| **수요 정보** | 구매자가 실제 수요/예산을 숨김 | 판매자가 낮은 가격으로 제안. 후회 | "우리의 실제 수요를 공유하면 더 좋은 솔루션이 가능하지 않을까?" |
| **능력·신뢰도 정보** | 일방이 과장 (사기 위험) | 합의 후 이행 실패. 제소 비용 | 제3자 검증 (실사, 레퍼런스) + 조건부 계약 |

### 정보 공개 순서의 중요성

**좋은 순서**: 자신의 정보 공개 → 상대도 공개 (Reciprocity 원칙)

**나쁜 순서**: 상대가 먼저 공개할 때까지 기다리기 → 상대도 기다림 → 교착

### 조건부 계약으로 정보 비대칭 극복

```
판매자 주장: "우리 제품의 품질은 매년 10% 향상됩니다"
구매자 의심: "그게 사실인가?"

해결책 — Contingent Contract:
"만약 실제로 10% 향상하면, 연간 $10만 추가 보상.
만약 5% 이하이면, 구매자가 $20만 환급받음."

→ 판매자도 본인 주장에 자신이 있으면 동의.
→ 거짓말의 비용이 높아짐.
```

## Schelling's Commitment Devices — 셸링의 약속 장치

### 정의

"자신의 선택 자유도를 자발적으로 제한하되, 그 제한이 상대에게 보이도록" 함으로써 협상 위치를 강화하는 기법.

### 2가지 핵심 요소

| 요소 | 의미 | 예 |
|------|------|------|
| **Irreversibility** (돌이킬 수 없음) | 한 번 약속하면 되돌릴 수 없어야 함 | "이 가격 이하로는 절대 팔 수 없습니다" (공개 선언) |
| **Visibility** (눈에 보임) | 상대방이 그 제한을 분명히 알아야 함 | 회의실 안에서 비밀리 하는 건 아무 효과 없음. 공개 약속, 3자 증인 필요 |

### 협상에서의 예

| 약속 장치 | 메커니즘 | 효과 | 위험 |
|----------|--------|------|------|
| **공개 입장 표명** | "우리는 이 가격으로 절대 움직일 수 없습니다"를 회의, SNS, 언론에 공개 | 상대가 "이건 진짜다"라고 믿고, 타협 제안 | "너무 경직됨" → 협상 자체 붕괴 |
| **대리인 위임** (Agent Restriction) | CEO가 "우리 담당자는 $100만까지만 권한 있음"이라고 공개 | 담당자가 "상급자 승인 필요"라고 주장 가능 → 시간 버는 동안 상대 생각 정리 | 상대가 "상급자를 데려와"라고 요구 |
| **매몰 비용(Sunk Cost) 신호** | 이미 상대에게 양보할 수 없는 상황을 만듦 | 예: "이미 이 조건으로 우리 직원들한테 약속했습니다" | 직원들이 실제로 그 약속에 의존하게 되면 신뢰도 상승. 거짓이면 발각 시 신뢰 붕괴 |
| **법적·계약적 제약** | 계약서에 "이 조건은 변경 불가"라고 명시 | 법적 강제력. 변경 시 위반금 부과 | 유연성 상실 |
| **제3자 증인/중재자** | "이 조건에 중재자가 동의했습니다" | 중립적 제3자의 입김 → 상대의 신뢰 상승 | 중재자 선택이 공정해야 함 |

### 효과

- **적절한 Commitment Device** → 상대가 "이건 진짜다"라고 믿음 → 협상 수렴 빠름
- **과도한 약속** → 상대가 "협상 자체를 포기하자"고 판단 → 교착 또는 결렬
- **거짓 약속** (실제로는 변경 가능한데 못 본척) → 발각 시 신뢰 완전 붕괴

## Commitment Tactics — 약속 전술 5가지

협상 힘을 강화하는 방법.

| 전술 | 메커니즘 | 적용 | 위험 |
|------|--------|------|------|
| **1. Public Position** | 자신의 입장을 대중·언론·상급자에게 선언 | "이 조건으로 수용할 수 없습니다"를 보도자료로 발표 | 협상 기회 상실 가능 |
| **2. Burning Bridges** | 다른 선택지를 의도적으로 제거 | "이 거래 아니면 사업 포기합니다"라고 공언 | 진짜 포기해야 할 수도 있음 |
| **3. Delegation with Restricted Authority** | 협상 권한을 제한된 범위로 위임 | "저는 이 범위까지만 결정할 수 있습니다. 상급자 결재 필요" | 상대가 "상급자를 데려와"라고 요구 → 시간 소모 |
| **4. Signaling Through Sunk Costs** | 이미 투자한 비용을 강조 | "우리는 이미 $50만을 이 프로젝트에 투자했습니다. 이 조건으로 수용해야만 수익화 가능합니다" | 상대가 "그건 우리 문제가 아니"라고 거부 |
| **5. Involving Third Parties** | 법률가, 중재자, 기관 등을 끌어들임 | "이 조건은 우리 법률자문가가 불가능하다고 했습니다" | 협상 복잡도 증가 |

### 실전 조합

**약한 BATNA를 강화하려면:**
- Public Position (비용 신호) + Delegation (시간 벌기) + Sunk Cost (투자 명시)

**강한 BATNA를 안전하게 활용:**
- 제3자 증인만 충분 (과도한 commitment 금지)

## Information Asymmetry Detection Protocol

협상 전에, 정보 비대칭 구조를 파악.

| 질문 | 비대칭 종류 | 대응 |
|------|-----------|------|
| **우리가 모르는 상대의 정보가 뭔가?** | 상대의 BATNA? 진짜 필요? 재정 상태? | 간접 질문으로 유추. 조건부 계약으로 risk 전가 |
| **상대가 모르는 우리 정보가 뭔가?** | 우리의 진짜 여유? 다른 대안? 최종 한계점? | 공개하면 협상 약화. 부분 공개로 신뢰 신호 |
| **누가 더 이 거래를 필요로 하는가?** | 시간 압박? 전략적 중요성? | 필요도 높은 쪽이 협상력 약함. 숨기기 필수 |
| **거짓말의 비용이 높은가?** | 제3자 검증 가능한가? 이행 감시 가능한가? | 검증 가능 → 거짓말 비용 높음 → 신뢰 향상. 검증 불가 → 조건부 계약 필수 |

## Game Theory Decision Protocol

모든 협상을 다음 체크리스트로 분석.

```
Step 1: 반복게임인가? (반복 확률 δ 추정)
  YES → Folk Theorem 작동. 미래 관계 가치 강조
  NO → 일회성. Nash 균형 주의. 구속력 있는 설계

Step 2: 정보 비대칭은?
  YES → 정보 공개 순서 결정. 조건부 계약 고려
  NO → 순수 이해관계 분배만 남음

Step 3: 약속 신뢰도는?
  낮음 → Commitment Device 강화 (공개 선언, 제3자 증인)
  높음 → 과도한 제약 금지 (유연성 유지)

Step 4: 내 BATNA가 충분한가?
  강함 → 높은 앵커링 가능. Commitment 현명하게 사용
  약함 → Commitment 신중히. "미래 거래 가능성" 강조

Step 5: 연합 형성 가능한가?
  예 → 여럿이 함께 협상 → 상대의 협상력 약화
  아니오 → 양자 협상. 상대의 제안에 유의

→ 이 5개 답을 바탕으로 협상 전략 조정
```
