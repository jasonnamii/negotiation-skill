# Advanced Frameworks — 고급 협상 이론

## Lax & Sebenius "3-D Negotiation" — 3차원 협상

협상은 테이블 위의 의사소통만 아니다. 테이블 설계, 사전 합의 구조, 참석자 선정이 결과의 60%를 결정한다.

### 3개 Dimension

| Dimension | 초점 | 의사결정 | 영향도 |
|-----------|------|--------|-------|
| **D1: Tactics (전술)** | 테이블 위의 상호작용. 앵커링, 양보, 감정 관리 | "우리가 뭘 말할 것인가" | 20~30% |
| **D2: Design (설계)** | 쟁점 구성, 다중 이슈 번들, 창의적 옵션 | "문제를 어떻게 구조화할 것인가" | 30~40% |
| **D3: Setup (셋업)** | 참석자, 진행 순서, 예비 협상, 우의적 기대 형성 | "누가, 언제, 어디서 협상하는가" | 30~40% |

### 각 Dimension의 구체 사례

**D1 Tactics (협상 테이블)**
```
우리: "$100만을 첫 제안으로 제시"
상대: "너무 높다. 우린 $60만만 가능"
우리: "왜 $60만인가?"
→ 상대의 비용/수익 구조 파악
→ 다른 쟁점으로 보상 가능한가?
```

**D2 Design (문제 재구조화)**
```
단순 설계: 가격 협상 → 단일 차원 (상대가 높을수록 우리는 저가 추구)
고급 설계: 가격 + 기간 + 서비스 번들
→ "가격 $70만 + 5년 계약 + 무료 유지보수" 
→ 상대가 장기 안정성 원함 → 초기 가격 낮춤 가능

또는:
→ 지급 방식 변경: "선금 없음, 실적 연동 지불"
→ 상대의 현금 흐름 제약 해결 → 가격 프리미엄 수용
```

**D3 Setup (테이블 전)**
```
나쁜 셋업: 상대가 주도권. CEO vs 담당자 (권력 비대칭). 한 번의 회의.
→ 상대의 조건이 대부분 관철됨

좋은 셋업: 
- 우리가 회의 장소·시간 선택 (방문 vs 초대)
- 사전 1:1 미팅으로 상대의 진짜 관심사 파악
- 양측 리더가 참석 (권력 균형)
- 3회 이상의 연쇄 회의 (첫 회는 "이해" 단계)
- 상대 내부 이해관계자(법무, 재무)와 사전 접촉

→ Setup 단계에서 이미 70~80% 합의된 상태로 테이블에 옴
```

### D1→D2→D3 순서의 중요성

**틀린 순서** (대부분의 협상):
```
D1 Tactics만 고민 (테이블에서 강하게 나가자)
→ D2 Design 이미 정해짐 (상대가 설정)
→ D3 Setup 이미 고착 (상대가 유리한 구조)
→ 결과: 패배
```

**올바른 순서**:
```
D3 Setup 먼저 (참석자, 순서, 장소, 사전 합의 구조)
→ D2 Design (다중 쟁점, 창의적 번들, 가능 범위 확장)
→ D1 Tactics 실행 (이미 70% 정해짐. 마무리만)
→ 결과: 승리
```

### 한국 협상에 적용

D3 Setup이 특히 중요:
- 회사 규모, 직급 대비 불균형 확인 (문제면 사전 조율)
- 사전 회담에서 "우리도 이 조건 수용 가능한가?"를 CEO에게 확인
- 이해관계자 동의 먼저 취득 (후속 "상급자 반대"로 번복 금지)

## Bazerman & Malhotra "Negotiation Genius" — 7 Investigative Principles

최고의 협상가는 무엇을 다르게 하는가?

### 7가지 원칙과 실행

| # | 원칙 | 일반 협상 | 천재 협상 |
|---|------|---------|---------|
| **1** | **"무엇"이 아닌 "왜" 물기** | "최선 가격이 뭔가요?" | "왜 정확히 그 가격인가요? 무엇을 그 가격으로 충당하려는 건가요?" |
| **2** | **요구 아닌 관심 화해** | "우린 $100만, 넌 $70만. 중간값 $85만" | "우리가 $100만이 필요한 이유 = 10명 팀 유지비. 넌 $70만이 필요한 이유 = 마진율 20% 필요. 다른 방식은?" |
| **3** | **흔하지 않은 동맹 찾기** | 경쟁사와는 협상 안 함 | "이 제품 기술을 X 회사도 필요로 하지 않을까? Y사와도 협상한다면 둘 다에게 나을 수 있을까?" |
| **4** | **요구를 기회로 해석** | 터무니없는 요구 → 거부 | "왜 이렇게 이상한 요구를 했을까? 어떤 미충족 need가 숨어있을까?" → 공동 이익 발견 |
| **5** | **상대 문제도 해결해주기** | "그건 상대의 문제" | "상대가 내부 승인을 못 받으면 이 협상도 구현 불가능. 상대가 자신의 상급자에게 설명할 근거를 주자" |
| **6** | **"No"를 최종으로 받지 않기** | 거부당하면 포기 | "왜 안 되나요? 어떤 조건이면 가능할까요? 우린 언제까지 기다릴 수 있고, 우린 어떻게 도와드릴 수 있을까요?" |
| **7** | **판매 vs 협상 구분** | 상대를 설득하려 함 (일방향) | 상대와 함께 문제를 풀려 함 (쌍방향) |

### 실전 팁

**원칙 1+2 조합: 깊이 있는 Interest 탐색**
```
상대: "우린 가격을 20% 낮춰야 한다"
우리(피상적): "그건 어렵습니다"
우리(천재): "20% 낮춰야 하는 이유가 뭔가요? 경영상 필요인가, 아니면 경쟁사 압력인가요?"
상대: "경쟁사가 18% 더 저렴하게 제시했습니다"
우리: "그럼 가격만 내리는 게 해결책이 아니라, 우리 제품의 unique value를 상대 구매자에게 설명하는 게 우선이지 않을까요? 함께 그걸 만들어보면?"
→ 가격 인하 없이 구매자 설득 구조로 전환
```

**원칙 5: 상대의 내부 문제 해결**
```
상대 구매자: "난 너희 조건에 동의하는데, 우리 CFO가 승인을 안 해줘"
우리(일반): "그건 니 문제다. 우린 이 조건으로 더 안 내린다"
우리(천재): "CFO 입장에서 뭐가 걱정이실까요? 혹시 우리가 문서를 다시 정리해서 CFO에게 설명할 자료를 만들어드리면 도움될까요?"
→ 상대의 내부 승인 프로세스도 함께 해결
```

## Kolb & Williams "Shadow Negotiation" — 숨겨진 협상

주 협상(공식) 뒤에는 "shadow negotiation"이 있다. 상대가 주장을 어떻게 수용하게 할 것인가의 싸움.

### 4가지 움직임 (Moves)

| Move | 정의 | 협상에서의 역할 |
|------|------|---------|
| **Positioning** | 상대와의 상대적 지위, 권력, 신뢰를 설정 | 누가 테이블을 주도하는가. "이건 내 제안"이라는 ownership 확보 |
| **Process** | 토론이 어떻게 진행되는가를 조형 | 의제, 시간, 참석자, 정보 공개 순서를 누가 정하는가 |
| **Power** | 상대가 협력 안 할 때의 영향력 행사 | "이 거래 안 하면 어떤 대가가 있는가"를 느끼게 함 |
| **Appreciative** | 상대의 기여, 관점, 우려를 인정 | "너의 의견도 중요하고, 내가 고려하고 있다"는 신호 |

### 4가지 전환 (Turns) — Shadow Negotiation에 대응하기

상대가 당신의 주장을 거부하거나 무시할 때:

| Turn | 대응 | 예 |
|------|------|------|
| **Correcting** | 상대의 잘못된 주장을 직접 정정 | 상대: "너희는 이전에 이 조건으로 수용했잖아" / 우리: "그건 다른 프로젝트였고, 이번은 다른 맥락입니다" |
| **Diverting** | 주제를 우리 유리한 방향으로 전환 | 상대가 가격만 고집 / 우리: "가격도 중요하지만, 이 계약이 3년 이후에도 sustainable한지가 더 중요하지 않을까요?" |
| **Interrupting** | 상대의 부당한 행동을 명시적으로 중단 시킴 | 상대가 계속 우리를 비난 / 우리: "미안하지만, 우리를 비난하는 방식으로는 이 문제를 풀 수 없습니다. 다시 시작할까요?" |
| **Naming** | 숨겨진 게임을 명시적으로 언급 | 상대가 암묵적으로 우리를 무시 / 우리: "지금까지 당신은 우리 제안에 한 번도 긍정적 응답을 하지 않으셨습니다. 우리의 뭐가 문제인가요?" |

### Shadow Negotiation 주의

- **Correcting/Interrupting 과다** → 상대가 "지배당한다"고 느낌 → 관계 악화
- **Appreciative 부족** → 상대가 무시받는다고 느낌 → 고착
- **Naming은 신중히** → 상대의 저항이 표면화될 수 있음 → 원하는 게 아니면 피하기

## BATNA 강화 전략 — 협상력의 기반

### BATNA 정의

"Best Alternative to Negotiated Agreement" = 협상이 실패했을 때 할 수 있는 최선의 선택지

### BATNA 강화 5단계

| 단계 | 실행 | 효과 |
|------|------|------|
| **1. 가능한 모든 대안 나열** | "이 협상이 안 되면 우리는 (a) 직접 개발, (b) 다른 공급업체, (c) 포기" | 넓은 시야 |
| **2. 각 대안의 현실성 평가** | "(a) 6개월·$200만 필요 / (b) 품질 70% / (c) 손실 $500만" | 객관적 판정 |
| **3. 최고 대안 상세 분석** | "(a)를 깊이 있게 분석. 정말 가능한가? 리스크는?" | 신뢰할 수 있는 BATNA |
| **4. BATNA 신뢰도 제3자 검증** | 변호사, 기술 고문, 재무 어드바이저가 "(a)가 현실적"이라고 확인 | 공신력 있는 BATNA |
| **5. 상대에게 신호** | 협상 중 "우리는 다른 옵션도 충분히 평가했습니다"라는 암시 (직접 언급 금지) | 상대의 협상력 약화 |

### BATNA 신호 방법

| 신호 방식 | 구체 | 효과 | 위험 |
|----------|------|------|------|
| **직접 언급** | "우린 X사와도 협상 중입니다" | 강함 | 상대가 "그럼 우린 포기한다"고 할 수도 |
| **간접 신호** | "우린 여러 옵션을 검토하고 있습니다" | 중간 | 해석의 여지. 상대가 무시할 수도 |
| **객관 근거** | "시장 벤치마크는 $80만입니다" | 강하면서도 중립적 | 객관 근거의 품질이 중요 |
| **제3자 언급** | "우리 법률자문가도 이 조건은 market standard라고 했습니다" | 강하면서도 일반화 | 제3자 신뢰도가 높아야 함 |

### 상대 BATNA 약화 (윤리적 범위 내)

| 기법 | 예 | 윤리성 |
|------|------|--------|
| **상대의 대안 수 줄이기** | "혹시 X사 제품도 검토하셨나요? 그쪽은 납기가 더 길어요" | 정보 제공. 합법 |
| **상대의 걷기(Walk-away)를 어렵게 하기** | "지금 이 프로젝트를 중단하면 팀 reputation에 타격이 될까봐요" (공감) | 현실을 지적. 합법 |
| **상대의 대안 비용 상승** | "다른 공급업체로 바꾸면 통합 비용이 3개월 추가로 들 겁니다" | 정보 제공. 합법 |
| **상대의 긴급성 악화** | "우리가 지금 못 하면 시장 기회가 3개월 후로 미뤄질 것 같아요" | 사실 제시. 합법 |

## Zone of Possible Agreement (ZOPA) 조작

ZOPA = 우리 유보가격(Walk-away) ~ 상대 유보가격(Walk-away).

### ZOPA 확대 (파이 늘리기)

| 기법 | 메커니즘 | 예 |
|------|---------|------|
| **시간 차원 추가** | "3년 계약으로 가격을 낮추고, 재계약 시 인상" | 상대: 초기 현금흐름 우려 해결. 우리: 장기 안정성 |
| **쟁점 추가** | "가격만 아니라 기술 지원, A/S도 포함" | 상대가 중요한 가치 추가. 가격 협상 여유 생김 |
| **비금전 가치 추가** | "가격은 높지만, 우리 CEO가 직접 컨설팅" | 상대에게 무형 가치 제공 |
| **조건부 조건** | "실적이 목표 달성하면 연간 보너스 30%" | 상대의 성과 동기 자극. 초기 비용 낮춤 |

### ZOPA 부재 시 해결책

```
우리 유보가격 $80만 (이 이하면 팔 수 없음)
상대 유보가격 $60만 (이 이상이면 못 뜸)
→ ZOPA 부재 ($20만 gap)

해결책:
1. 파이 확대 (위의 4가지 기법)
2. BATNA 강화 (우리: $80만으로 내리기 or 상대: $60만으로 올리기)
3. 협상 포기 (합의 불가능)
```
