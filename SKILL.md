# Negotiation Skill — 협상 설계 엔진

person-profiler가 "상대를 아는 것", biz-skill이 "전략을 짜는 것"이라면, 이 스킬은 **"테이블 위에서 어떻게 주고받을 것인가"를 5-Layer 깊이로 구조화하는 것**.

협상은 준비의 질이 결과의 80%를 결정한다(HBS PON). 이 스킬은 그 준비를 프로토콜화하되, 심리학·게임이론·문화차이·프로세스 설계를 체계적으로 통합한다.

## §1. 모드 자동 판별

입력 유형모드예시앞으로의 협상 상황**설계**"이번 텀시트 협상 어떻게 준비하지"진행 중 교착**돌파**"협상이 막혔어", "상대가 안 움직여", "교착 상황"완료된 협상**복기**"이번 딜 결과 분석해줘"

복합 시 설계→돌파 파이프라인. 애매하면 명확화 질문.

## §2. 5-Layer Analysis Architecture

협상의 성패는 5개 층이 동시에 작동할 때 결정된다. 각 층은 독립적이지만 상호 제약한다.

Layer핵심 질문출처스크리닝**L1: Interest**양측의 진짜 이해관계·동기·우선순위는? Position vs Interest 분리 가능한가?Fisher & Ury, Bazerman 7원리● 필수 / ○ 관련 / - 무관**L2: Structure**BATNA·ZOPA·제안 공간을 정의하는 구조는? 3-D 협상(Setup·Design·Tactics) 프레임에서 어느 차원이 가장 영향력 있는가?Lax & Sebenius, Raiffa● 필수 / ○ 관련 / - 무관**L3: Psychology**상대방을 지배하는 심리 메커니즘은 무엇인가? (앵커링·반응 평가절하·손실회피·공정성 규범 중)Tversky, Kahneman, Cialdini● 필수 / ○ 관련 / - 무관**L4: Relationship**문화적 맥락·권력 역학·신뢰 수준이 어떤 제약을 만드는가?Hofstede, Meyer, Shadow Negotiation● 필수 / ○ 관련 / - 무관**L5: Execution**합의 후 이행 구조·조건부 계약·프로세스 설계는?Raiffa, Game Theory Folk Theorem● 필수 / ○ 관련 / - 무관

**Layer 상호 작용 규칙**:

- L1 우선: Interest를 구명하지 않고 L2 구조 설계 금지
- L3는 L2 전술 선택을 통제: 앵커링 약한 상대에게는 앵커링 전술 無효
- L4는 L3 심리 기법을 제약: 문화적 비위험 회피 문화에서 강한 앵커링은 관계 파괴
- L5는 모든 층을 구현: 아무리 좋은 전략도 이행 구조가 없으면 무의미

## §3. 심리 프로필 스크리닝

상대방이 어떤 심리 메커니즘에 가장 강하게 반응하는가를 판정 → 전술 개인화.

심리 메커니즘신호전술 응답**앵커링 취약**첫 제안에 "그게 우리 가격이 맞아?"라고 선뜻 따라감공격적 앵커링 활용 + 근거 제시 (약함)**반응 평가절하**"상대가 제안했다고 좋은 건 아니야"라는 고정관념 반복제3자 재제시 또는 근거-우선 프레이밍**손실회피 강함**"현재 상태 유지"를 강하게 선호, 변화 제안 거부손실 프레이밍 ("지금 안 하면 기회상실")**공정성 규범 민감**"같이 50:50 나눠야지"라는 표현 빈번공정성 기준을 먼저 합의하고 적용**지위·위신**자신의 위치·권력 표시 빈번상대 체면 보장 장치 (숨겨진 양보 전술)

→ 분석 깊이: references/negotiation-psychology.md

## §4. 게임이론 의무 체크

모든 협상 전략을 게임이론 렌즈로 재검토. 3가지 핵심 질문:

질문찾는 것전술 조정**반복게임인가**?일회성 vs 지속 관계반복 → 협력 인센티브 강화 / 일회성 → 배팅 전술 OK**정보 비대칭**?누가 더 많이 알고 있는가정보 공개 전술, 조건부 계약 활용**약속 가능한가**?상대방 약속 신뢰도, 제3자 집행 메커니즘약속 불가 → 구속력 있는 조건부 계약 필수

## §5. 실행 파이프라인

### 설계 모드

```
①상황 구조화 → ②5-Layer 스크리닝 → ③심리 프로필 분석 → ④게임이론 검증
→ ⑤Deep Layer 분석(활성화된 층만) → ⑥전략 설계 → ⑦3-시나리오 시뮬레이션 → ⑧액션 플랜
```

**①상황 구조화**: 협상 대상·상대방·쟁점·배경·제약·시간압박·관계 맥락 추출.

**②5-Layer 스크리닝** (각 층을 ● 핵심 / ○ 관련 / - 무관으로 1줄 판정):

Layer판정핵심 질문L1 Interest● / ○ / -L2 Structure● / ○ / -L3 Psychology● / ○ / -L4 Relationship● / ○ / -L5 Execution● / ○ / -

**③심리 프로필 분석** (●로 판정된 층에서):

- 상대방의 주요 심리 트리거 (앵커링/반응평가절하/손실회피/공정성/지위)
- 각 트리거의 강도 (강·중·약)
- 이에 따른 전술 선택

**④게임이론 검증**:

- 반복게임 여부 + 할인인수 추정 (얼마나 미래를 중시하는가)
- 정보 비대칭 구조 (누가 뭘 숨기고 있는가)
- 약속 실행 메커니즘 (자발적 vs 구속력 있는 계약)

**⑤Deep Layer 분석** (활성화 층만):

- L1 ● → 이해관계 매핑 (→ references/negotiation-theories.md §Fisher & Ury)
- L2 ● → BATNA·ZOPA·3-D 협상 (→ references/advanced-frameworks.md)
- L3 ● → 심리 전술 설계 (→ references/negotiation-psychology.md)
- L4 ● → 문화·권력 분석 (→ references/cultural-negotiation.md)
- L5 ● → 프로세스 설계 (→ references/process-design.md)

**⑥전략 설계**:

전략 요소설계 내용BATNA·ZOPA 매핑양측 대안·유보가격·목표점·앵커앵커링 전술첫 제안 금액/조건 + 근거 프레이밍 (상대 심리 프로필 기반)양보 시퀀스양보 항목·교환 비율·양보 곡선 (체감형)패키지 딜로그롤링 (다중 쟁점 번들)감정·관계 관리체면 보장 장치·문화적 민감성

**⑦3-시나리오 시뮬레이션**:

시나리오상대 반응우리 대응예상 결과L3 심리 기전최선기본최악

각 시나리오에 If-Then 매핑. L3 심리 메커니즘이 반응을 어떻게 예측 가능하게 하는지 명시.

**⑧액션 플랜**: 협상 전·중·후 체크리스트 + 응급 대응.
